## 一、项目概述

TTS_MultiModel 是一个**双引擎语音合成平台**，集成 Qwen3-TTS 与 VoxCPM2 两大语音模型，提供四合一功能：声音设计（捏人）、语音克隆、精品音色、剧本工坊。

**核心特性**:
- 完全离线、解压即用、无需安装环境
- CUDA Graph 加速推理
- 双模型规格自由切换 (1.7B高保真 / 0.6B极速版)
- 低显存优化 (6G-8GB即可运行)

## 九、技术亮点与风险

### 9.1 技术亮点

1. **CUDA Graph 加速**: faster-qwen3-tts 通过手动 CUDA 图捕获大幅提升推理速度
2. **绿色免安装**: WinPython 便携版实现解压即用
3. **双引擎架构**: Qwen3-TTS + VoxCPM2 覆盖TTS和VC场景
4. **低显存优化**: 动态模型卸载，6G-8GB显存即可运行
5. **完全离线**: 所有模型本地缓存，无需网络

### 9.2 技术风险

| 风险类型 | 风险项 | 影响 | 建议 |
|---------|--------|------|------|
| **安全风险** | SSL全局禁用 | HTTPS请求无验证 | 仅针对特定域名禁用 |
| **维护风险** | integrated_app.py 单文件过大 | 难以维护和扩展 | 拆分为多模块 |
| **兼容性风险** | torch版本要求>=2.5.1 (faster-qwen3-tts) | 可能与requirements.txt的>=2.1.0冲突 | 统一版本要求 |
| **依赖风险** | faster-qwen3-tts 社区项目 | 长期维护不确定 | 关注上游更新 |
| **并发风险** | 全局模型状态无锁保护 | 并发请求可能冲突 | 添加读写锁 |

## 十、总结

TTS_MultiModel 项目采用 **Python 3.12 + PyTorch + Gradio + FastAPI** 技术栈，集成 **Qwen3-TTS** 和 **VoxCPM2** 两大开源语音模型，通过 **faster-qwen3-tts** 实现 CUDA Graph 加速。

项目以**绿色便携 Python (WinPython)** 为核心，实现完全离线、解压即用的桌面级语音合成工作站。适合个人使用场景，技术选型合理，但存在 SSL 安全隐患和单文件维护困难等问题。
